🔌 API ENDPOINTS (56+):

Doctors:
  GET    /api/doctors              - Список врачей
  GET    /api/doctors/{id}         - Врач по ID
  POST   /api/doctors              - Создать
  PUT    /api/doctors/{id}         - Обновить
  DELETE /api/doctors/{id}         - Удалить

Patients:
  GET    /api/patients             - Список пациентов
  GET    /api/patients/{id}        - Пациент по ID
  POST   /api/patients             - Создать
  PUT    /api/patients/{id}        - Обновить
  DELETE /api/patients/{id}        - Удалить

Appointments:
  GET    /api/appointments         - Список приемов
  GET    /api/appointments/{id}    - Прием по ID
  POST   /api/appointments         - Создать
  PUT    /api/appointments/{id}    - Обновить
  DELETE /api/appointments/{id}    - Удалить
  GET    /api/appointments/doctor/{id}    - Приемы врача
  GET    /api/appointments/patient/{id}   - Приемы пациента

Services:
  GET    /api/services             - Список услуг
  GET    /api/services/{id}        - Услуга по ID
  POST   /api/services             - Создать
  PUT    /api/services/{id}        - Обновить
  DELETE /api/services/{id}        - Удалить


💾 БАЗА ДАННЫХ:

  Файл: backend/medical_center.db
  Тип: SQLite (автоматическое создание)
  
  Таблицы:
    • doctors      - Врачи (id, name, specialization, email, phone, ...)
    • patients     - Пациенты (id, first_name, last_name, email, phone, ...)
    • appointments - Приемы (id, patient_id, doctor_id, appointment_date, ...)
    • services     - Услуги (id, name, description, price, ...)
